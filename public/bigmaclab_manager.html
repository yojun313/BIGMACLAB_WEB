1. DATABASE

1.1 개요

DATABASE 섹션은 BIGMACLAB MANAGER에서 데이터베이스와 관련된 모든 작업을 처리하는 곳입니다. 이 섹션을 통해 사용자는 데이터베이스를 불러오고, 갱신하며, 필요한 데이터를 쉽게 조회할 수 있습니다.

1.2 사용 방법

DATABASE 섹션의 각 기능은 다음과 같은 절차로 사용할 수 있습니다:

  • 데이터베이스 불러오기: 프로그램을 실행하면 자동으로 데이터베이스를 불러옵니다. 데이터를 처음부터 다시 불러오고 싶다면, '새로고침' 버튼을 클릭하세요.
  • 데이터베이스 갱신: 데이터베이스에 변경 사항이 있을 때 '새로고침' 버튼을 클릭하면 목록이 갱신됩니다.
  • 데이터베이스 상세 보기: 테이블에서 특정 데이터베이스를 클릭하면, 상세 정보를 볼 수 있는 '세부 정보' 버튼이 활성화됩니다. 이 버튼을 클릭하면 선택한 데이터베이스의 상세 내용을 볼 수 있는 팝업 창이 열립니다.

1.2.1 새로고침 버튼

버튼을 클릭하면 데이터베이스 목록이 최신 상태로 갱신됩니다. 이 버튼을 자주 사용하여 데이터베이스의 최신 상태를 유지하십시오.

1.2.2 세부 정보 버튼

버튼은 선택된 데이터베이스의 상세 정보를 보여줍니다. 이 기능은 데이터베이스의 특정 항목을 확인해야 할 때 유용합니다.

1.3 주의사항

주의: 데이터베이스 갱신 중 프로그램을 종료하지 마십시오. 이로 인해 데이터가 손상될 수 있습니다. 데이터베이스를 갱신할 때는 모든 작업을 저장하고 프로그램이 정상적으로 종료될 때까지 기다려야 합니다.

2. CRAWLER

2.1 개요

CRAWLER 섹션은 웹사이트에서 자동으로 데이터를 수집하는 웹 크롤러를 관리하는 곳입니다. 이 기능을 통해 지정된 웹사이트에서 정보를 수집하고, 이를 데이터베이스에 저장할 수 있습니다.

2.2 사용 방법

CRAWLER 섹션에서는 다음과 같은 작업을 수행할 수 있습니다:

  • 크롤러 실행: 크롤러를 실행하여 데이터를 수집하려면, '크롤러 실행' 버튼을 클릭하세요. 이 버튼을 클릭하면 지정된 URL에서 데이터 수집이 시작됩니다.
  • 수집된 데이터 보기: 크롤링 작업이 완료되면, '수집된 데이터 보기' 버튼을 클릭하여 수집된 데이터를 확인할 수 있습니다.

2.2.1 크롤러 실행 버튼

버튼을 클릭하면 크롤링 작업이 시작됩니다. 크롤링이 시작되면, 크롤러는 지정된 웹사이트에서 데이터를 수집하며, 완료되면 알림이 표시됩니다.

2.2.2 수집된 데이터 보기 버튼

버튼을 클릭하면, 크롤링된 데이터를 테이블 형식으로 확인할 수 있습니다. 이 기능을 통해 수집된 데이터의 내용을 쉽게 파악할 수 있습니다.

2.3 주의사항

주의: 크롤러가 실행되는 동안 프로그램을 종료하거나 인터넷 연결을 끊지 마세요. 크롤러가 데이터를 수집하는 중에 이러한 작업을 수행하면 데이터가 손실되거나 작업이 중단될 수 있습니다.

3. ANALYSIS

3.1 개요

ANALYSIS 섹션은 데이터를 분석하고 그 결과를 시각화할 수 있는 도구를 제공합니다. 이 섹션을 사용하면 수집된 데이터를 기반으로 유용한 인사이트를 도출할 수 있습니다.

3.2 사용 방법

ANALYSIS 섹션의 주요 기능을 사용하는 방법은 다음과 같습니다:

  • 데이터 로드: 분석할 데이터를 로드하려면 '데이터 로드' 버튼을 클릭하세요. 파일 선택 창이 열리면 분석할 데이터를 선택할 수 있습니다.
  • 데이터 분석: 데이터를 로드한 후 '분석 실행' 버튼을 클릭하여 분석을 시작할 수 있습니다.
  • 결과 시각화: 분석이 완료되면 '결과 시각화' 버튼을 클릭하여 분석 결과를 그래프나 표 형태로 확인할 수 있습니다.

3.2.1 데이터 로드 버튼

버튼을 클릭하면 분석할 데이터를 불러올 수 있습니다. 이 기능은 분석 작업의 첫 번째 단계입니다.

3.2.2 분석 실행 버튼

버튼을 클릭하면 로드된 데이터를 바탕으로 분석이 수행됩니다. 분석이 진행되는 동안 프로그램이 응답하지 않는 것처럼 보일 수 있으나, 이는 정상적인 동작입니다.

3.2.3 결과 시각화 버튼

버튼을 클릭하면 분석 결과를 다양한 형태로 시각화할 수 있습니다. 그래프나 표로 표시된 결과를 통해 데이터의 의미를 쉽게 파악할 수 있습니다.

3.3 주의사항

주의: 분석할 데이터의 양이 많을 경우, 분석 시간이 길어질 수 있습니다. 프로그램이 응답하지 않는 것처럼 보일 수 있으니, 완료될 때까지 기다려 주세요.

4. BOARD

4.1 개요

BOARD 섹션은 프로그램의 버전 관리와 버그 리포팅을 위한 기능을 제공합니다. 사용자는 이 섹션에서 새로운 프로그램 버전을 추가하거나, 발견된 버그를 기록하고 수정된 내용을 확인할 수 있습니다.

4.2 사용 방법

BOARD 섹션에서 수행할 수 있는 작업은 다음과 같습니다:

  • 버전 관리: 프로그램의 버전을 추가, 수정, 삭제할 수 있습니다.
  • 버그 리포팅: 프로그램 사용 중 발견된 버그를 기록하고 수정된 내용을 관리할 수 있습니다.

4.2.1 버전 추가 버튼

버튼을 클릭하면 새로운 프로그램 버전을 추가할 수 있는 창이 열립니다. 여기서 버전 번호, 릴리스 날짜, 변경 사항 등을 입력할 수 있습니다.

4.2.2 버전 삭제 버튼

버튼을 클릭하면 선택된 버전을 삭제할 수 있습니다. 삭제된 버전은 복구할 수 없으니 신중하게 사용하세요.

4.2.3 버그 추가 버튼

버튼을 클릭하면 발견된 버그를 기록할 수 있는 창이 열립니다. 여기서 버그의 제목, 상세 내용을 입력할 수 있습니다.

4.2.4 버그 삭제 버튼

버튼을 클릭하면 선택된 버그 리포트를 삭제할 수 있습니다. 이 기능도 신중하게 사용해야 합니다.

4.3 주의사항

주의: 버전 삭제 및 버그 삭제 작업은 되돌릴 수 없습니다. 삭제 전에 충분히 검토하고 작업을 수행하세요.

5. WEB

5.1 개요

WEB 섹션은 프로그램 내에서 웹사이트를 탐색하거나 특정 URL을 방문할 수 있는 기능을 제공합니다. 사용자는 이 섹션을 통해 다양한 웹 페이지에 접근하고, 필요에 따라 관련 정보를 빠르게 확인할 수 있습니다.

5.2 사용 방법

WEB 섹션의 주요 기능은 다음과 같습니다:

  • 웹사이트 탐색: 프로그램 내에서 여러 웹사이트를 탐색할 수 있습니다.
  • URL 열기: 특정 URL을 입력하여 해당 페이지를 직접 열 수 있습니다.

5.2.1 웹사이트 탐색 버튼

버튼을 클릭하면 기본 설정된 웹사이트가 열립니다. 사용자는 이 창에서 다양한 페이지를 탐색할 수 있습니다.

5.2.2 URL 열기 버튼

버튼을 클릭하면 사용자 지정 URL을 입력할 수 있는 창이 열립니다. URL을 입력하고 확인을 누르면 해당 웹페이지가 열립니다.

5.3 주의사항

주의: 웹사이트 탐색 중 인터넷 연결이 불안정하면 페이지 로딩에 문제가 발생할 수 있습니다. 안정적인 네트워크 환경에서 사용하세요.

6. USER

6.1 개요

USER 섹션은 사용자 관리와 관련된 모든 작업을 처리하는 곳입니다. 이 섹션에서 사용자를 추가, 삭제하거나 사용자 정보를 수정할 수 있습니다. 관리자는 이 섹션을 통해 사용자 계정을 관리하고, 프로그램 사용을 모니터링할 수 있습니다.

6.2 사용 방법

USER 섹션의 주요 작업은 다음과 같습니다:

  • 사용자 추가: 새로운 사용자를 추가하려면 '사용자 추가' 버튼을 클릭하세요. 사용자의 이름, 이메일, 푸쉬 알림 키 등을 입력할 수 있는 창이 열립니다.
  • 사용자 삭제: 기존 사용자를 삭제하려면 '사용자 삭제' 버튼을 클릭하세요. 삭제된 사용자는 복구할 수 없습니다.
  • 사용자 정보 수정: 사용자의 정보(이메일, 푸쉬 알림 키 등)를 수정하려면 해당 사용자를 선택하고 '수정' 버튼을 클릭하세요.

6.2.1 사용자 추가 버튼

버튼을 클릭하면 새로운 사용자를 등록할 수 있는 창이 열립니다. 이 창에서 사용자의 이름, 이메일, 푸쉬 알림 키 등을 입력할 수 있습니다.

6.2.2 사용자 삭제 버튼

버튼을 클릭하면 선택된 사용자를 삭제할 수 있습니다. 이 작업은 되돌릴 수 없으므로 신중히 결정하세요.

6.2.3 사용자 정보 수정 버튼

버튼을 클릭하면 선택된 사용자의 정보를 수정할 수 있습니다. 이메일 주소나 푸쉬 알림 키 등을 변경할 수 있습니다.

6.3 주의사항

주의: 사용자 삭제 작업은 되돌릴 수 없으며, 삭제된 사용자의 데이터는 모두 사라집니다. 삭제 전에 반드시 확인하십시오.
