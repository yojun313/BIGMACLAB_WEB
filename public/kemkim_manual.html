KEM KIM 분석 과정에 대한 설명

1. 데이터 로딩

CSV 파일에서 텍스트 데이터를 불러옵니다. 이 데이터는 주로 뉴스 기사나 문서의 텍스트와 각 문서의 날짜 정보로 구성되어 있습니다. 분석의 시작 단계로, 모든 이후 단계의 기초가 됩니다.

2. 데이터 전처리

불러온 데이터를 분석에 적합한 형태로 변환하고 그룹화하는 과정입니다. 이 과정은 다음과 같은 단계를 포함합니다:

  • 날짜 변환: 데이터의 날짜 정보를 표준화된 형식(예: YYYY-MM-DD)으로 변환합니다. 이는 데이터가 일관되게 처리되도록 하기 위한 중요한 단계입니다.
  • 기간 그룹화: 데이터를 사용자가 지정한 기간(예: 월, 분기, 연도)으로 그룹화합니다. 예를 들어, 연도별로 데이터를 그룹화하면, 각 연도별로 키워드 분석을 수행할 수 있게 됩니다.
  • 빈 데이터 처리: 그룹화된 데이터에서 빈 데이터(예: 특정 기간에 데이터가 없는 경우)를 처리하여 분석에 포함되지 않도록 합니다.

3. 키워드 추출

텍스트 데이터에서 빈도가 높은 단어들을 추출하여 키워드로 지정합니다. 이 단계에서는 다음과 같은 작업이 이루어집니다:

  • 빈도 계산: 각 기간마다 텍스트 데이터에서 자주 등장하는 단어들의 빈도를 계산합니다. 이 계산은 다음 수식을 사용합니다:
    • 단어 빈도(기간) = 해당 기간에 등장한 특정 단어의 총 횟수
  • 공통 키워드 선택: 모든 기간에 걸쳐 공통으로 등장하는 상위 빈도 단어들을 분석 대상으로 선택합니다. 예를 들어, 특정 단어가 모든 연도에서 등장했다면, 이 단어는 공통 키워드로 선정됩니다.
  • 제외 단어 목록: 분석에서 제외할 단어 목록을 생성할 수 있습니다. 이는 노이즈가 될 수 있는 단어를 걸러내기 위한 단계입니다.

4. 분석

각 기간별로 키워드의 중요성을 분석합니다. 주요 분석 단계는 다음과 같습니다:

  • 단어 빈도 계산: 각 기간마다 키워드의 등장 횟수를 계산합니다. 이 값은 단어 빈도로 정의되며, 특정 기간에 해당 단어가 얼마나 자주 등장했는지를 나타냅니다.
  • 문서 빈도 계산: 키워드가 등장한 문서의 수를 계산하여, 문서 빈도로 정의합니다. 이 값은 해당 기간 동안 얼마나 많은 문서에서 해당 키워드가 언급되었는지를 보여줍니다.
  • 증가율 계산: 키워드의 시간에 따른 증가율을 계산합니다. 이 계산은 DoV(Time-Weighted 증가율)과 DoD(Time-Weighted 증가율)로 이루어지며, 다음과 같은 수식을 사용합니다:
    • DoV = (단어 빈도 / 전체 문서 수) × (1 - 가중치 × 시간 지수)
    • DoD = (문서 빈도 / 전체 문서 수) × (1 - 가중치 × 시간 지수)
    • 여기서, 시간 지수는 현재 기간과 첫 번째 기간 사이의 차이입니다. 이를 통해 각 키워드가 시간에 따라 얼마나 중요하게 변화했는지를 평가합니다.
  • 시그널 분석: 키워드의 중요도에 따라 네 가지 유형으로 분류합니다:
    • 강한 신호(Strong Signal): 높은 단어 빈도와 높은 증가율을 가진 키워드.
    • 약한 신호(Weak Signal): 낮은 단어 빈도와 높은 증가율을 가진 키워드.
    • 잠재 신호(Latent Signal): 낮은 단어 빈도와 낮은 증가율을 가진 키워드.
    • 잘 알려진 신호(Well-Known Signal): 높은 단어 빈도와 낮은 증가율을 가진 키워드.

5. 시각화

분석된 데이터를 그래프로 시각화하여, 각 키워드가 어떤 성격을 가지는지 한눈에 파악할 수 있습니다. 시각화 단계는 다음과 같습니다:

  • 사분면 분석: 각 키워드의 평균 빈도와 증가율을 기준으로, 키워드를 네 개의 사분면 중 하나에 배치합니다. 이때 사용되는 기준은 다음과 같습니다:
    • x축(단어 빈도) = 평균 단어 빈도 또는 사용자 지정 값
    • y축(증가율) = 평균 증가율 또는 사용자 지정 값
  • 그래프 생성: 각 키워드의 위치를 그래프 상에 표시합니다. 이 그래프는 네 개의 사분면으로 나뉘어 있으며, 각 사분면은 키워드의 성격을 나타냅니다. 예를 들어, 특정 키워드가 강한 신호라면 그래프의 1사분면에 배치됩니다.
  • 좌표 데이터 저장: 각 키워드의 좌표 데이터를 CSV 파일로 저장하여, 이후 분석에 활용할 수 있도록 합니다. 좌표 데이터는 각 키워드가 그래프 상에서 어떤 위치에 있었는지를 나타냅니다.

6. 결과 저장

분석의 최종 결과는 다양한 형식으로 저장됩니다. 다음은 생성되는 파일과 디렉토리에 대한 설명입니다:

  • 빈도 데이터 저장:

    각 키워드의 단어 빈도와 문서 빈도 데이터를 CSV 파일로 저장합니다. 이 파일들은 다음과 같은 디렉토리에 저장됩니다:

    • TF 디렉토리: 각 기간별로 키워드의 단어 빈도를 나타내는 CSV 파일이 저장됩니다. 파일명은 TF_YYYY.csv 형식으로 저장되며, 여기서 YYYY는 해당 기간을 나타냅니다.
    • DF 디렉토리: 각 기간별로 키워드의 문서 빈도를 나타내는 CSV 파일이 저장됩니다. 파일명은 DF_YYYY.csv 형식으로 저장되며, 여기서 YYYY는 해당 기간을 나타냅니다.
  • 증가율 데이터 저장:

    키워드의 시간에 따른 증가율 데이터를 CSV 파일로 저장합니다. 이 파일들은 다음과 같은 디렉토리에 저장됩니다:

    • DoV 디렉토리: 각 기간별로 키워드의 DoV(Time-Weighted 증가율) 값을 나타내는 CSV 파일이 저장됩니다. 파일명은 DoV_YYYY.csv 형식입니다.
    • DoD 디렉토리: 각 기간별로 키워드의 DoD(Time-Weighted 증가율) 값을 나타내는 CSV 파일이 저장됩니다. 파일명은 DoD_YYYY.csv 형식입니다.
  • 시그널 데이터 저장:

    키워드를 신호의 강도에 따라 분류한 데이터를 CSV 파일로 저장합니다. 강한 신호, 약한 신호, 잠재 신호, 잘 알려진 신호 등으로 분류된 키워드들이 저장되며, 다음과 같은 파일이 생성됩니다:

    • DoV_signal.csv: 각 키워드가 DoV 기준으로 분류된 결과를 포함합니다.
    • DoD_signal.csv: 각 키워드가 DoD 기준으로 분류된 결과를 포함합니다.
    • Final_signal.csv: DoV와 DoD를 종합하여 최종적으로 분류된 신호 데이터를 저장합니다.
  • 그래프 저장:

    분석 결과를 시각화한 그래프를 PNG 파일로 저장합니다. 이 파일들은 다음과 같은 이름으로 저장됩니다:

    • TF_DOV_graph.png: 단어 빈도와 DoV 증가율을 나타낸 그래프가 저장됩니다. 이 그래프는 각 키워드가 시간에 따라 어떻게 변화했는지를 시각적으로 보여줍니다.
    • DF_DOD_graph.png: 문서 빈도와 DoD 증가율을 나타낸 그래프가 저장됩니다. 이 그래프는 키워드가 얼마나 많은 문서에서 언급되었는지를 시각적으로 표현합니다.
    • KEM_graph.png: 최종적으로 추출된 강한 신호를 중심으로 그린 그래프가 저장됩니다. 이 그래프는 분석된 데이터 중에서 가장 중요한 키워드를 보여줍니다.
    • KIM_graph.png: 최종적으로 추출된 중요한 신호를 중심으로 그린 그래프가 저장됩니다. 이 그래프는 중요한 키워드가 시간에 따라 어떻게 변화했는지를 보여줍니다.
  • 애니메이션 저장:

    키워드가 시간에 따라 어떻게 변화했는지를 나타내는 애니메이션 GIF 파일을 생성할 수 있습니다. 이 파일은 keyword_movements.gif로 저장됩니다. 애니메이션은 각 기간별로 키워드의 변화 과정을 시각적으로 표현하며, 특정 기간 동안 키워드가 어떻게 이동하고 변화했는지를 명확하게 보여줍니다.

중요 사항

데이터 양: 데이터의 양이 많을 경우, 분석 시간이 길어질 수 있습니다. 분석 과정이 진행되는 동안 프로그램이 응답하지 않는 것처럼 보일 수 있으나, 이는 정상적인 동작입니다.

제외 단어: 특정 단어를 분석에서 제외할 수 있으며, 이를 통해 분석의 정확성을 높일 수 있습니다. 제외된 단어들은 filtered_words.csv 파일에 기록되어 나중에 참고할 수 있습니다.

결과 해석: 최종 결과물은 키워드의 등장 빈도와 시간에 따른 중요성 변화를 나타내며, 이를 통해 다양한 인사이트를 얻을 수 있습니다. 예를 들어, 특정 키워드가 특정 기간에 급격히 중요해졌다면, 해당 기간에 해당 키워드와 관련된 중요한 사건이 있었음을 추정할 수 있습니다.
