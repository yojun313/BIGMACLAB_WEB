KEM KIM 분석 과정에 대한 상세 설명

1. 데이터 로딩

분석의 첫 단계로, CSV 파일에서 텍스트 데이터를 불러옵니다. 이 데이터는 데이터의 텍스트와 각 문서의 날짜 정보로 구성됩니다.

2. 데이터 전처리

불러온 데이터를 분석에 적합한 형태로 변환하고, 그룹화하는 과정입니다. 이 단계에서는 다음과 같은 작업이 이루어집니다:

  • 날짜 변환: 데이터의 날짜 정보를 표준화된 형식(예: YYYY-MM-DD)으로 변환합니다.
  • 기간 그룹화: 데이터를 사용자가 지정한 기간(예: 월, 분기, 연도)으로 그룹화합니다. 예를 들어, 연도별로 데이터를 그룹화하면, 각 연도별로 키워드 분석을 수행할 수 있습니다.

3. 키워드 추출

텍스트 데이터에서 빈도가 높은 단어들을 추출하여 키워드로 지정합니다. 이 과정에서 다음 작업이 포함됩니다:

  • 빈도 계산: 각 기간마다 텍스트 데이터에서 자주 등장하는 단어들의 빈도를 계산합니다.
    • 단어 빈도(기간) = 해당 기간에 등장한 특정 단어의 총 횟수
  • 공통 키워드 선택: 모든 기간에 걸쳐 공통으로 등장하는 상위 빈도 단어들을 분석 대상으로 선택합니다. 상위 단어 개수는 KEM KIM 실행 시 "상위 단어 개수"로 조정할 수 있습니다
  • 제외 단어 목록: 분석에서 제외할 단어 목록을 생성할 수 있습니다. 이 과정은 노이즈를 제거하여 분석의 정확성을 높이기 위한 것입니다.

4. 분석

각 기간별로 키워드의 중요성을 분석합니다. 주요 분석 단계는 다음과 같습니다:

  • 단어 빈도 계산: 각 기간마다 키워드의 등장 횟수를 계산합니다. 이 값은 단어 빈도로 정의되며, 특정 기간에 해당 단어가 얼마나 자주 등장했는지를 나타냅니다.
  • 문서 빈도 계산: 키워드가 등장한 문서의 수를 계산하여 문서 빈도로 정의합니다. 이 값은 해당 기간 동안 얼마나 많은 문서에서 해당 키워드가 언급되었는지를 보여줍니다. 이 작업은 cal_df 메서드로 수행됩니다.
  • 증가율 계산: 키워드의 시간에 따른 증가율을 계산합니다. 이 계산은 DoV(Time-Weighted 증가율)과 DoD(Time-Weighted 증가율)로 이루어지며, 다음과 같은 수식을 사용합니다:
    • DoV = (단어 빈도 / 전체 문서 수) × (1 - 가중치 × 시간 지수)
    • DoD = (문서 빈도 / 전체 문서 수) × (1 - 가중치 × 시간 지수)
  • 시그널 분석: 키워드의 중요도에 따라 네 가지 유형으로 분류합니다:
    • 강한 신호(Strong Signal): 높은 단어 빈도와 높은 증가율을 가진 키워드입니다.
    • 약한 신호(Weak Signal): 낮은 단어 빈도와 높은 증가율을 가진 키워드입니다.
    • 잠재 신호(Latent Signal): 낮은 단어 빈도와 낮은 증가율을 가진 키워드입니다.
    • 잘 알려진 신호(Well-Known Signal): 높은 단어 빈도와 낮은 증가율을 가진 키워드입니다.

5. 시각화 및 데이터 저장

분석된 데이터를 그래프로 시각화하고, 결과를 다양한 형식으로 저장합니다. 주요 단계는 다음과 같습니다:

  • 사분면 분석: 각 키워드의 평균 빈도와 증가율을 기준으로 키워드를 네 개의 사분면 중 하나에 배치합니다. 이 작업은 _analyze_signals 메서드로 처리됩니다.
  • 그래프 생성: 각 키워드의 위치를 그래프 상에 표시합니다. 그래프는 네 개의 사분면으로 나뉘어 있으며, 각 사분면은 키워드의 특성을 나타냅니다.
  • 결과 저장: 단어 빈도, 문서 빈도, DoV, DoD 등 분석 결과를 CSV 파일로 저장합니다. 또한, 분석 결과를 시각화한 그래프를 PNG 파일로 저장합니다.

6. 추적 데이터 생성

키워드별로 기간을 따라 신호 변화를 추적합니다. 이 과정은 각 키워드의 시계열적 중요성 변화를 파악하는 데 중점을 둡니다:

  • DoV 및 DoD 신호 추적: 각 기간 동안 키워드의 DoV 및 DoD 신호를 추적하여, 키워드가 시간에 따라 어떻게 변화했는지를 분석합니다. 이 작업은 trace_keyword_positions 메서드를 통해 이루어집니다.
  • 신호 변화 기록: 각 키워드의 신호 변화를 기록하고, 이 데이터를 CSV 파일로 저장합니다. 이 과정은 키워드의 시간에 따른 변화를 명확히 이해하는 데 도움이 됩니다.

7. 시그널 필터링 및 좌표 데이터 업데이트

추적된 데이터에서 시계 방향으로 신호 이동이 일어나지 않은 키워드를 걸러내고, 해당 키워드를 좌표 데이터에서 제거합니다:

  • 시그널 필터링: 키워드의 신호 이동이 시계 방향으로 진행되지 않은 경우, 해당 키워드를 필터링합니다. 이 작업은 filter_clockwise_movements 메서드를 통해 이루어집니다.
  • 좌표 데이터 업데이트: 필터링된 키워드를 기반으로 좌표 데이터를 업데이트합니다. 이로 인해, 불필요한 데이터가 제거되고, 분석의 정확성이 높아집니다.
  • 최종 좌표 데이터 저장: 필터링된 좌표 데이터를 CSV 파일로 저장하여, 최종 분석 결과에 반영됩니다.

8. 최종 결과물 생성 및 저장

최종적으로 필터링된 데이터를 기반으로 KEM KIM 분석 결과를 생성하고, 결과를 저장합니다:

1. 최상위 폴더: kimkem_folder_path

최종 분석 결과가 저장되는 최상위 폴더로, 폴더명은 CSV 파일 이름과 분석 기간, 분석 시간에 기반하여 생성됩니다.

2. Data 폴더

주기별로 계산된 TF, DF, DoV, DoD 데이터를 저장합니다.

  • TF/YYYY_TF.csv: 각 기간에 대해 키워드의 단어 빈도(Term Frequency) 데이터가 저장됩니다.
  • DF/YYYY_DF.csv: 각 기간에 대해 키워드의 문서 빈도(Document Frequency) 데이터가 저장됩니다.
  • DoV/YYYY_DoV.csv: 각 기간에 대해 키워드의 DoV(Time-Weighted 증가율) 데이터가 저장됩니다.
  • DoD/YYYY_DoD.csv: 각 기간에 대해 키워드의 DoD(Time-Weighted 증가율) 데이터가 저장됩니다.

3. Result 폴더

최종 분석 결과와 시각화 자료를 저장합니다.

  • Graph: 최종 분석 결과를 바탕으로 생성된 그래프 이미지 파일이 저장됩니다.
  • Signal: 최종 신호 분석 결과가 CSV 파일로 저장됩니다.

4. Trace 폴더

추적 분석을 위한 중간 결과와 최종 결과를 저장합니다.

  • Trace_Result: 각 기간별로 신호 분석 결과와 그래프 이미지 파일이 저장됩니다.
  • Trace_Data: 추적 분석을 위해 DoV 및 DoD 데이터가 저장됩니다.
    • DoV: 각 기간별로 키워드의 DoV 데이터가 저장됩니다.
    • DoD: 각 기간별로 키워드의 DoD 데이터가 저장됩니다.

중요 사항

데이터 양: 데이터의 양이 많을 경우, 분석 시간이 길어질 수 있습니다. 이 과정에서 프로그램이 응답하지 않는 것처럼 보일 수 있으나, 이는 정상적인 동작입니다.

제외 단어: 분석에서 제외할 단어들을 지정할 수 있으며, 이들은 filtered_words.csv 파일에 기록됩니다. 이는 분석의 정확성을 높이기 위한 과정입니다.

결과 해석: 최종 결과는 키워드의 등장 빈도와 시간에 따른 중요성 변화를 나타냅니다. 이를 통해 특정 키워드가 특정 기간에 급격히 중요해졌다면, 해당 기간에 관련된 중요한 사건이 있었음을 추정할 수 있습니다.
